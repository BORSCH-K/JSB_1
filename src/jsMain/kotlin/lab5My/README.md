**Маршрутизация** – одна из ключевых возможностей фреймворка React, позволяющая управлять переходами и отображением различных компонентов в зависимости от текущего пути URL. Без маршрутизации веб-приложение на React было бы просто набором изолированных компонентов, не способных связываться друг с другом и предлагать пользователю удобную навигацию.

В react'е два основных провайдера: `HashRouter` и `BroweserRouter`. Они отличаются представлением маршрутов.     
В `HashRouter`: имя_сервера/#/маршрут. Работает только локально.   
В `BroweserRouter`: имя_сервера/адрес_внутри_приложения. Но тогда после перехода по ссылке, на сервер придет запрос => нужна настройка сервера.  

`HashRouter` –  это подтип компонента `<Router>`, который использует значение хэша для обновления пользовательского интерфейса приложения на основе изменений URL. Компонент получает свое название, потому что он использует символ хэша. Все, что следует за этим символом, будет проигнорировано в запросе сервера.

~ *Сервер отвечает только за главную страницу, а "подстраницы" задача для передаваемого html-документа. Т.е. для перехода на страницу не нужен запрос серверу.* ~

`Routers` говорит о том, что внутри компоненты, меняющиеся от значения маршрута. То есть "дальше маршруты".

`Router` – это компонент верхнего уровня с отслеживанием состояния, который заставляет работать все остальные компоненты навигации и хуки.

Параметры маршрута:  
path (*путь*) - значение после решетки. :название_маршрута.  
element - готовый элемент  отображения.

У react router основной источник - его внутреннее состояние.

```Kotlin
Routes { // маршруты  
    Route {  // маршрут
        path = ":name" // имя для параметра
        element = 
	        Element.create { // уже готовыйй элемент
	            this.name = linkName // имя страницы, linkName - состояние
	             // this - Route, маршрут
	             // this.name - параметр маршрута
	        } 
    }}
```

```Kotlin
Routes{
	courses.map{ course ->
		Route{ // маршрут
			path = course.name // готовое имя маршрута
			element = Element.create{ // элемент
				c = course
			}
		}}}
```

Функция **useParams()** возвращает набор параметров маршрута. Фактически такой набор представляет объект, а каждый отдельный параметр - свойство этого объекта.

```Kotlin
val params: Record<String, String> = useParams()
```

~ *Record представляет параметры в виде пары key-value* ~
